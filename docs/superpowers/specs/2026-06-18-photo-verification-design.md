# Дизайн: Фото-верификация выполненных работ

**Дата:** 2026-06-18  
**Статус:** Утверждён  
**Ветка:** `feature/photo-verification`

---

## Проблема

1. Верификация выполненных работ — фиктивная: собственник нажимает кнопку без каких-либо доказательств.
2. Баг в ролях: кнопка "Верифицировать" показывается начальнику производства (`isPM`), а не собственнику. Собственник видит "—".

## Цель

Дать начальнику производства возможность прикрепить фото при отметке работы как выполненной. Собственник видит фото в истории и верифицирует на основе реального доказательства.

## Скоуп (эта ветка)

- Загрузка одного фото при отметке "Выполнено" (необязательно)
- Сжатие фото на клиенте до ~300–500KB
- Хранение в Supabase Storage
- Просмотр фото в модалке в истории
- Исправление бага: верификация только для собственника

## Вне скоупа (отдельная ветка)

- Мобильная оптимизация интерфейса (responsive layout, крупные touch-элементы)

## Архитектура

### 1. База данных

```sql
ALTER TABLE maintenance_logs ADD COLUMN photo_url TEXT;
```

Никаких новых таблиц. URL фото хранится прямо в `maintenance_logs`.

### 2. Supabase Storage

Bucket: `maintenance-photos`

| Параметр | Значение |
|----------|----------|
| Публичность | Публичный (чтение без авторизации) |
| Запись | Только `production_manager` (RLS) |
| Макс. размер | 10MB (до сжатия) |
| Форматы | image/jpeg, image/png, image/webp |
| Путь файла | `{userId}/{logId}.jpg` |

RLS политика на загрузку:
```sql
CREATE POLICY "pm_can_upload_photos"
ON storage.objects FOR INSERT TO authenticated
WITH CHECK (
  bucket_id = 'maintenance-photos' AND
  EXISTS (
    SELECT 1 FROM profiles
    WHERE profiles.id = auth.uid()
    AND profiles.role = 'production_manager'
  )
);

CREATE POLICY "authenticated_can_read_photos"
ON storage.objects FOR SELECT TO authenticated
USING (bucket_id = 'maintenance-photos');
```

### 3. ChecklistClient.tsx

**Новое состояние:**
```ts
const [photos, setPhotos] = useState<Record<string, File | null>>({})
const [photoPreviews, setPhotoPreviews] = useState<Record<string, string>>({})
```

**Сжатие на клиенте (Canvas API):**
- Масштабировать до max 1200px по длинной стороне
- JPEG, quality 0.8
- Результат: ~200–400KB

**UI (появляется только при `decision === 'done'`):**

Состояние A — фото не выбрано:
```
[✓ Выполнено]  [Пропустить]
[📷 Прикрепить фото]
```

Состояние B — фото выбрано (предпросмотр):
```
[✓ Выполнено]  [Пропустить]
┌─────────────────────────────┐
│  [изображение 100% ширины,  │
│   max-height: 200px,        │
│   object-fit: contain]      │
└─────────────────────────────┘
[🔄 Заменить фото]  [✕ Удалить]
```

Предпросмотр достаточно крупный чтобы оценить чёткость. НП может нажать "Заменить фото" (открывает file picker снова) или "Удалить" (убирает фото, возвращает к состоянию A). Оба действия доступны **только до нажатия "Сохранить"**.

После нажатия "Сохранить" — фото загружается в Storage и URL записывается в БД. Изменение фото после сабмита невозможно (нет UI, нет RLS на UPDATE для photo_url).

**Логика handleSubmit:**
1. Для каждого лога с фото: сжать → загрузить в Storage → получить публичный URL
2. Upsert в maintenance_logs с photo_url
3. Если загрузка фото упала — показать ошибку, лог не сохраняется

### 4. HistoryClient.tsx

**Исправление бага ролей:**

```tsx
// Было:
isPM ? <button>Верифицировать</button> : <span>—</span>

// Стало:
isOwner ? <button>Верифицировать</button> : <span>—</span>
```

Пропс `isPM` заменяется на `isOwner` (передаётся из `history/page.tsx`).

**Колонка "Верификация" (новая логика):**

```
log.verified = true  →  "✓ Верифицировано" (для всех)
log.verified = false:
  + photo_url есть  →  иконка 🖼 (кликабельная) + кнопка "Верифицировать" (только owner)
  + photo_url нет   →  кнопка "Верифицировать" (только owner) / "—" (для НП)
```

**Модальное окно:**
- Overlay с `z-50`, клик вне фото закрывает
- ESC закрывает
- Фото: `max-h-[80vh] object-contain`
- Кнопка "Верифицировать" под фото (только для собственника)
- Кнопка "Закрыть"

## Обработка ошибок

| Сценарий | Поведение |
|----------|-----------|
| Фото не выбрано | Лог сохраняется без photo_url (фото необязательно) |
| Ошибка загрузки фото в Storage | Inline-ошибка под кнопкой "Сохранить", лог не пишется |
| Фото > 10MB (до сжатия) | Ошибка "Файл слишком большой, максимум 10MB" |
| Не-изображение | input accept="image/*" блокирует на уровне браузера |
| Ошибка верификации | Inline-ошибка в таблице истории (уже есть) |
